Memo: My Office VMware Setting

2012-06-18

My office is using the environment of the WindowsXP. Developing with node.js was inconvenient things on Windows XP. Therefore, recently, I began to develop in linux on VMware. Here, I’ll describe (for myself) how to build environment.

My office environment

  • Windows XP (32bits version)
  • CPU : core i3 370M
  • to connect internet through firewall

To build environment

  • install(upgrade) VMware player 4.04 (upgrade last week)
  • install Fedora 17 64bits KDE to VMware
    • recently,my favorite desktop is KDE
    • if CPU is 64bits, Guest OS can select 64bits OS
  • proxy setting
    • create the following files in /etc/profile.d, and then this will work in any shell for any user of the system

  • update fedora $ sudo yum -y update

  • setup Japanese environment $ sudo yum -y groupinstall “Japanese Support”

      * groupinstall is so convenient, and I install any groups
    • create or edit following files

  • setup Development environment

    • $ sudo yum -y groupinstall “Development Libraries”
    • $ sudo yum -y groupinstall “Development Tools”
    • $ sudo yum -y install kate my favorite editor :)

  • setup vmware-tools

    • VMware menu : [Vertural machine(V)] -> [VMware tools install] select
    • select and extract VMware Tools (mount as CD device)
    • to go extracting directory and install $ sudo ./vmware-install.pl

  • setup git (proxy)

  • setup node.js

  • setup npm proxy

  • setup mongodb
